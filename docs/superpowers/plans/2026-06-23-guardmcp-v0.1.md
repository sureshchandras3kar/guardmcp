# GuardMCP v0.1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a policy-enforced MongoDB MCP server that acts as a secure gateway between AI clients and MongoDB — "AI proposes, GuardMCP decides."

**Architecture:** FastMCP (mcp 1.27.1) handles the MCP protocol layer (stdio + SSE); a `GuardPipeline` class chains policy evaluation → risk classification → approval gate → execution → field masking before any MongoDB query runs. A separate FastAPI app on a second port exposes approval REST endpoints for human operators.

**Tech Stack:** Python 3.12+, mcp[fastmcp] 1.27.1, FastAPI 0.116+, Pydantic v2, pydantic-settings, Motor 3.7+, PyYAML, uvicorn, pytest + pytest-asyncio + mongomock-motor, Docker

## Deliverable 4: Policy Engine Design

**Evaluation order (deny-wins at each step):**

```
1. mode == readonly  AND  action is write op  →  DENIED
2. collection in collections.deny             →  DENIED
3. collections.allow non-empty
   AND collection NOT in collections.allow    →  DENIED
4. action.value in actions.deny               →  DENIED
5. actions.allow non-empty
   AND action.value NOT in actions.allow      →  DENIED
6. risk == CRITICAL AND approval.critical     →  APPROVAL_REQUIRED
7. risk == HIGH     AND approval.high         →  APPROVAL_REQUIRED
8. default                                    →  ALLOWED
```

**Design assumptions challenged:**
- Spec has both allow and deny lists for collections/actions. Deny always wins (explicit deny beats allow). This prevents misconfiguration where an item appears in both.
- `mode: readonly` is redundant with `actions.deny: [insert, update, delete]` but kept as a fail-safe. Both checks run independently.
- Agent identity comes from the `agent` parameter in each tool call — not from the MCP session. This is intentional: MCP does not natively authenticate callers (v0.1). Future: derive agent from JWT/mTLS client cert.

## Deliverable 5: MCP Integration Design

**Library:** `mcp[fastmcp]` 1.27.1 — `FastMCP` high-level API.

**Transports supported:**
- `stdio` — for Claude Desktop (spawned subprocess model)
- `sse` — for HTTP-based clients (Cursor, OpenAI Agents)
- `streamable-http` — newer MCP clients

**Tool naming convention:** `mongodb_{action}` (e.g. `mongodb_find`, `mongodb_aggregate`)

**Agent identity:** v0.1 passes `agent` as an explicit tool parameter. This is a known limitation — the MCP protocol does not authenticate callers. Mitigations in roadmap: v0.3 adds JWT bearer tokens verified at tool-call time.

**Error handling:** All tool functions catch exceptions and return `{"status": "error", "reason": str(e)}` rather than propagating. This prevents MCP protocol teardown on DB errors.

**Tool parameters are typed Pydantic models** — FastMCP auto-generates `inputSchema` JSON Schema from type annotations.

## Deliverable 6: Security Review

### What GuardMCP gets right
- Policy evaluation happens **before** MongoDB I/O — no TOCTOU window
- `Action` is an enum — no string comparison, no injection via action name
- `max_find_limit` caps document exfiltration per query
- Field masking runs on **all** result paths — masker cannot be bypassed
- Audit log is append-only with asyncio lock — no lost entries under concurrency

### Known weaknesses in v0.1 (with mitigations)

| Risk | Severity | Mitigation |
|------|----------|------------|
| Agent identity is self-declared in tool params | HIGH | v0.3: JWT auth; until then, document this clearly |
| MongoDB filter allows `$where` / `$function` (JS execution) | HIGH | Add query sanitizer: reject any key starting with `$` that is in a blocklist |
| Audit log is local file — can be deleted | MEDIUM | Ship with log rotation config; v0.4: structured log sink (Loki/CloudWatch) |
| Approval store is in-memory — lost on restart | MEDIUM | Pending approvals lost; operators must re-trigger; v0.2: Redis-backed store |
| No rate limiting | MEDIUM | v0.3: per-agent request rate limits in policy |
| No pipeline size limit on aggregate | MEDIUM | Add `max_pipeline_stages` config; default 10 |
| Policy loaded once at startup | LOW | v0.2: `SIGHUP` hot-reload |
| `$lookup` can JOIN to non-permitted collections | MEDIUM | Inspect pipeline stages; deny `$lookup` on disallowed collections |

### Recommended additions to v0.1
1. **Query sanitizer** — block `$where`, `$function`, `$accumulator` before execution
2. **Pipeline stage inspector** — for aggregate, check `$lookup.from` against collection policy
3. **Max result size** — reject results > configurable byte threshold

## Deliverable 7: Threat Model

**Trust boundary:** GuardMCP sits between untrusted AI clients and trusted MongoDB.

```
THREAT: Prompt injection via crafted query
VECTOR: AI client passes {"$where": "function(){sleep(10000)}"} as filter
IMPACT: DoS, JS execution in MongoDB
MITIGATION: Block $where/$function in query sanitizer (add to v0.1)

THREAT: Collection enumeration via error messages
VECTOR: AI probes collection names; "collection X not in allowed list" reveals policy
IMPACT: Information disclosure
MITIGATION: Return generic "access denied" — do not reveal whether collection exists

THREAT: Data exfiltration via large limit
VECTOR: AI passes limit=999999
IMPACT: Full collection dump
MITIGATION: max_find_limit cap enforced by MongoExecutor (not caller-configurable)

THREAT: Audit log bypass
VECTOR: Crash between execution and log write
IMPACT: Unlogged operations
MITIGATION: Log BEFORE execution for deny/approval; log after for allowed (accepted gap)

THREAT: Pipeline $lookup bypass
VECTOR: aggregate with $lookup to denied collection
IMPACT: Access to collections not in allow list
MITIGATION: Inspect $lookup.from in pipeline stages against collection policy

THREAT: Policy file tampering
VECTOR: Attacker modifies policy YAML on disk
IMPACT: Elevated permissions
MITIGATION: File permissions 640 root:guardmcp; checksum validation on load (v0.2)

THREAT: Approval replay
VECTOR: Approval_id reused or brute-forced
IMPACT: Unauthorized execution
MITIGATION: UUIDs are 122-bit random; resolved approvals cannot be re-approved (status check)

THREAT: MongoDB credential exposure
VECTOR: mongodb_uri leaked in logs/errors
IMPACT: Direct DB access
MITIGATION: Never log mongodb_uri; pydantic-settings reads from env (not config files)
```

## Deliverable 8: Risk Engine Design

**Principle:** deterministic lookup, no LLM, no heuristics.

```python
Action.FIND        → LOW
Action.COUNT       → LOW
Action.AGGREGATE   → MEDIUM    # can compute derived data; $lookup broadens scope
Action.INSERT_ONE  → MEDIUM    # adds data; reversible with delete
Action.UPDATE_ONE  → HIGH      # modifies existing data
Action.UPDATE_MANY → HIGH      # blast radius: many documents
Action.DELETE_ONE  → HIGH      # data loss; reversible only with backup
Action.DELETE_MANY → CRITICAL  # mass data loss
Action.DROP        → CRITICAL  # collection destroyed
```

**Configurability (v0.2):** The risk map is extracted to `risk_levels` in the policy YAML:

```yaml
# Future: per-policy risk overrides
risk_levels:
  aggregate: HIGH   # org wants tighter control on aggregation
  insert_one: HIGH
```

In v0.1, the map is a module-level dict in `risk/engine.py`. Changing risk levels = edit + redeploy. This is intentional simplicity.

## Deliverable 9: Approval Workflow Design

```
MCP Tool Call
    │
    ▼
GuardPipeline.run()
    │
    ├─ PolicyEngine returns APPROVAL_REQUIRED
    │
    ▼
ApprovalStore.create()           → creates ApprovalRequest(status="pending")
    │                               creates asyncio.Event for this ID
    │
    ▼
ApprovalStore.wait_for_decision()  → awaits asyncio.Event with timeout=300s
    │
    │ (meanwhile, human operator polls GET /approvals/pending)
    │ (human POSTs /approvals/{id}/decide {"approved": true})
    │ (ApprovalStore.resolve() sets status + fires asyncio.Event)
    │
    ├─ status == "approved"  →  execute MongoDB operation
    └─ status == "denied" or timeout  →  return {"status":"denied","reason":"..."}
```

**Key design decisions:**
- `asyncio.Event` — zero-overhead polling; operator decision immediately unblocks the coroutine
- Timeout defaults to 300s — prevents MCP clients from hanging indefinitely
- Approval requests survive the await — operators can query GET /approvals/{id} for history
- In-memory store: approved requests are not re-executable (resolved status gate)
- v0.2: webhook notification to Slack/PagerDuty when approval_required

## Deliverable 10: Async Performance Considerations

1. **Motor is natively async** — all MongoDB I/O is non-blocking; no thread pools needed
2. **AuditLogger uses asyncio.Lock** — prevents interleaved JSONL lines under concurrent requests
3. **ApprovalStore.wait_for_decision uses asyncio.wait_for** — does not block the event loop
4. **FastMCP is built on anyio** — compatible with asyncio and trio; no blocking calls in tool handlers
5. **PolicyLoader loads at startup** — zero I/O per request for policy lookup (dict.get)
6. **FieldMasker is pure Python** — no I/O; runs in microseconds per document
7. **MongoExecutor.execute uses match/case** — O(1) dispatch, no if-chain overhead
8. **No shared mutable state between requests** except ApprovalStore (protected by asyncio primitives)

**Bottleneck:** MongoDB I/O. GuardMCP adds <1ms overhead per request (policy eval + masking). For high throughput, MongoDB connection pool tuning (`maxPoolSize` in Motor) is the primary lever.

**Caveat on approval workflow:** `wait_for_decision` holds an MCP connection open for up to 300s. MCP clients must support long-lived tool calls. Claude Desktop does; others may timeout. Document this.

All tools return JSON strings. All accept `agent: str` and `collection: str`.

```
mongodb_find(agent, collection, filter?, limit?, projection?) → JSON
mongodb_count(agent, collection, filter?) → JSON
mongodb_aggregate(agent, collection, pipeline_stages) → JSON
mongodb_insert_one(agent, collection, document) → JSON
mongodb_update_one(agent, collection, filter, update) → JSON
mongodb_update_many(agent, collection, filter, update) → JSON
mongodb_delete_one(agent, collection, filter) → JSON
mongodb_delete_many(agent, collection, filter) → JSON
```

**Success response:**
```json
{"status": "success", "data": [...]}
```

**Denied response:**
```json
{"status": "denied", "reason": "collection 'payroll' is not permitted"}
```

**Approval required (after timeout or denial):**
```json
{"status": "denied", "reason": "approval denied or timed out"}
```

### Approval REST API

```
GET  /approvals/pending
     → 200 [ApprovalRequest, ...]

GET  /approvals/{approval_id}
     → 200 ApprovalRequest
     → 404 {"detail": "approval not found"}

POST /approvals/{approval_id}/decide
     Body: {"approved": true|false}
     → 200 {"approval_id": "...", "status": "approved"|"denied"}
     → 404 {"detail": "approval not found or already resolved"}
```

## Architecture Challenges & Improvements

**Challenge 1: Agent identity is self-declared**
Spec says agent name comes in the tool call. Any AI can claim to be "ops-bot". This is the highest-risk design decision in v0.1. Recommendation: document this prominently in the README as a known limitation. v0.3 fixes it with JWT. Until then, GuardMCP is appropriate for trusted internal AI clients, not public-facing deployments.

**Challenge 2: Approval workflow blocks MCP connections**
A 300s wait inside a tool call is unusual. Most MCP clients were designed for sub-second tool execution. Alternatives: (a) return immediately with `{"status": "pending", "approval_id": "..."}` and require the client to poll a `get_approval_status` tool; (b) use SSE push notifications. The blocking approach is simpler for v0.1 — document the limitation and recommend SSE transport for approval-heavy policies.

**Challenge 3: YAML policy is per-agent, not per-operation**
The spec allows `actions.allow: [find, aggregate]` but you cannot say "allow aggregate only on the `analytics` collection." This is a coarse-grained model. v0.3 should introduce collection-action matrix policies. For v0.1, agents needing fine-grained control should use the deny list to block specific operations per collection.

**Challenge 4: Field masking is shallow (top-level only)**
The spec masks `email`, `password` at the document root. Nested fields like `user.email` or arrays of objects are not masked. Recommend: dot-notation support in mask_fields (`user.email`, `contacts[].phone`). Implement in v0.1 as it's a security issue if documents have nested PII.

```python
from pydantic import BaseModel, model_validator


class CollectionPolicy(BaseModel):
    allow: list[str] = []
    deny: list[str] = []

    @model_validator(mode="after")
    def no_allow_deny_overlap(self) -> "CollectionPolicy":
        overlap = set(self.allow) & set(self.deny)
        if overlap:
            raise ValueError(f"collections in both allow and deny: {overlap}")
        return self


class ActionPolicy(BaseModel):
    allow: list[str] = []
    deny: list[str] = []


class ApprovalPolicy(BaseModel):
    high: bool = False
    critical: bool = False


class Policy(BaseModel):
    agent: str
    mode: str = "readonly"
    collections: CollectionPolicy = CollectionPolicy()
    actions: ActionPolicy = ActionPolicy()
    mask_fields: list[str] = []
```

```python
# src/guardmcp/policy/loader.py
from pathlib import Path
import yaml
from .models import Policy


class PolicyLoader:
    def __init__(self, policy_path: Path) -> None:
        self._path = policy_path
        self._policies: dict[str, Policy] = {}

    def load(self) -> None:
        with open(self._path) as f:
            data = yaml.safe_load(f)

        if isinstance(data, list):
            items = data
        elif isinstance(data, dict) and "agents" in data:
            items = data["agents"]
        else:
            items = [data]

        self._policies = {
            (p := Policy.model_validate(item)).agent: p
            for item in items
        }

    def get(self, agent: str) -> Policy | None:
        return self._policies.get(agent)

    def all(self) -> list[Policy]:
        return list(self._policies.values())
```

```python
# src/guardmcp/policy/engine.py
from ..models.domain import Action, Decision, DecisionStatus, Request, RiskLevel
from .models import Policy

_WRITE_ACTIONS = frozenset({
    Action.INSERT_ONE,
    Action.UPDATE_ONE,
    Action.UPDATE_MANY,
    Action.DELETE_ONE,
    Action.DELETE_MANY,
    Action.DROP,
})


class PolicyEngine:
    def evaluate(self, request: Request, policy: Policy, risk: RiskLevel) -> Decision:
        if policy.mode == "readonly" and request.action in _WRITE_ACTIONS:
            return Decision(
                status=DecisionStatus.DENIED,
                reason=f"action '{request.action.value}' blocked in readonly mode",
                risk=risk,
            )

        if request.collection in policy.collections.deny:
            return Decision(
                status=DecisionStatus.DENIED,
                reason=f"collection '{request.collection}' is not permitted",
                risk=risk,
            )

        if policy.collections.allow and request.collection not in policy.collections.allow:
            return Decision(
                status=DecisionStatus.DENIED,
                reason=f"collection '{request.collection}' is not permitted",
                risk=risk,
            )

        if request.action.value in policy.actions.deny:
            return Decision(
                status=DecisionStatus.DENIED,
                reason=f"action '{request.action.value}' is explicitly denied",
                risk=risk,
            )

        if policy.actions.allow and request.action.value not in policy.actions.allow:
            return Decision(
                status=DecisionStatus.DENIED,
                reason=f"action '{request.action.value}' is not in the allowed list",
                risk=risk,
            )

        if risk == RiskLevel.CRITICAL and policy.approval.critical:
            return Decision(
                status=DecisionStatus.APPROVAL_REQUIRED,
                reason=f"action '{request.action.value}' is classified as CRITICAL risk",
                risk=risk,
            )

        if risk == RiskLevel.HIGH and policy.approval.high:
            return Decision(
                status=DecisionStatus.APPROVAL_REQUIRED,
                reason=f"action '{request.action.value}' is classified as HIGH risk",
                risk=risk,
            )

        return Decision(
            status=DecisionStatus.ALLOWED,
            reason="request permitted by policy",
            risk=risk,
        )
```

```python
# src/guardmcp/masking/masker.py
from typing import Any

MASK_VALUE = "***masked***"


class FieldMasker:
    def __init__(self, fields: list[str]) -> None:
        self._fields = frozenset(fields)

    def mask(self, doc: dict[str, Any]) -> dict[str, Any]:
        return {
            k: MASK_VALUE if k in self._fields else v
            for k, v in doc.items()
        }

    def mask_result(self, result: Any) -> Any:
        if not self._fields:
            return result
        if isinstance(result, list):
            return [self.mask(item) if isinstance(item, dict) else item for item in result]
        if isinstance(result, dict):
            return self.mask(result)
        return result
```

```python
import asyncio
import json
from datetime import datetime, timezone
from pathlib import Path
from ..models.domain import AuditRecord


class AuditLogger:
    def __init__(self, log_path: Path) -> None:
        self._path = log_path
        self._path.parent.mkdir(parents=True, exist_ok=True)
        self._lock = asyncio.Lock()

    async def log(self, record: AuditRecord) -> None:
        line = json.dumps(record.model_dump(mode="json")) + "\n"
        async with self._lock:
            with open(self._path, "a") as f:
                f.write(line)

    def build_record(
        self,
        agent: str,
        collection: str,
        action: str,
        status: str,
        reason: str | None = None,
        risk: str | None = None,
        request_id: str | None = None,
    ) -> AuditRecord:
        return AuditRecord(
            timestamp=datetime.now(timezone.utc),
            agent=agent,
            collection=collection,
            action=action,
            status=status,
            reason=reason,
            risk=risk,
            request_id=request_id,
```

```python
# src/guardmcp/approval/store.py
import asyncio
from datetime import datetime, timezone
from uuid import uuid4
from typing import Any
from .models import ApprovalRequest
from ..models.domain import RiskLevel


class ApprovalStore:
    def __init__(self, timeout_seconds: float = 300.0) -> None:
        self._timeout = timeout_seconds
        self._store: dict[str, ApprovalRequest] = {}
        self._events: dict[str, asyncio.Event] = {}

    def create(
        self,
        agent: str,
        action: str,
        collection: str,
        risk: RiskLevel,
        reason: str,
        params: dict[str, Any],
    ) -> ApprovalRequest:
        req = ApprovalRequest(
            approval_id=str(uuid4()),
            agent=agent,
            action=action,
            collection=collection,
            risk=risk,
            reason=reason,
            params=params,
            created_at=datetime.now(timezone.utc),
        )
        self._store[req.approval_id] = req
        self._events[req.approval_id] = asyncio.Event()
        return req

    def get(self, approval_id: str) -> ApprovalRequest | None:
        return self._store.get(approval_id)

    def list_pending(self) -> list[ApprovalRequest]:
        return [r for r in self._store.values() if r.status == "pending"]

    def resolve(self, approval_id: str, approved: bool) -> ApprovalRequest | None:
        req = self._store.get(approval_id)
        if req is None or req.status != "pending":
            return None
        req.status = "approved" if approved else "denied"
        event = self._events.get(approval_id)
        if event:
            event.set()
        return req

    async def wait_for_decision(self, approval_id: str) -> str:
        event = self._events.get(approval_id)
        if event is None:
            return "denied"
        try:
            await asyncio.wait_for(event.wait(), timeout=self._timeout)
        except asyncio.TimeoutError:
            req = self._store.get(approval_id)
            if req and req.status == "pending":
                req.status = "denied"
            return "denied"
        req = self._store.get(approval_id)
        return req.status if req else "denied"
```

```python
# src/guardmcp/mongo/executor.py
from typing import Any
from bson import ObjectId
from .client import MongoClient
from ..models.domain import Action


def _serialize(obj: Any) -> Any:
    if isinstance(obj, ObjectId):
        return str(obj)
    if isinstance(obj, list):
        return [_serialize(i) for i in obj]
    if isinstance(obj, dict):
        return {k: _serialize(v) for k, v in obj.items()}
    return obj


class MongoExecutor:
    def __init__(self, client: MongoClient, max_limit: int = 1000) -> None:
        self._client = client
        self._max_limit = max_limit

    async def execute(self, collection: str, action: Action, params: dict[str, Any]) -> Any:
        col = self._client.get_collection(collection)

        match action:
            case Action.FIND:
                filter_ = params.get("filter", {})
                limit = min(int(params.get("limit", 100)), self._max_limit)
                projection = params.get("projection")
                docs = await col.find(filter_, projection).to_list(limit)
                return _serialize(docs)

            case Action.COUNT:
                return await col.count_documents(params.get("filter", {}))

            case Action.AGGREGATE:
                pipeline = params.get("pipeline", [])
                docs = await col.aggregate(pipeline).to_list(None)
                return _serialize(docs)

            case Action.INSERT_ONE:
                result = await col.insert_one(params.get("document", {}))
                return {"inserted_id": str(result.inserted_id)}

            case Action.UPDATE_ONE:
                result = await col.update_one(params["filter"], params["update"])
                return {"matched": result.matched_count, "modified": result.modified_count}

            case Action.UPDATE_MANY:
                result = await col.update_many(params["filter"], params["update"])
                return {"matched": result.matched_count, "modified": result.modified_count}

            case Action.DELETE_ONE:
                result = await col.delete_one(params["filter"])
                return {"deleted": result.deleted_count}

            case Action.DELETE_MANY:
                result = await col.delete_many(params["filter"])
                return {"deleted": result.deleted_count}

            case Action.DROP:
                await col.drop()
                return {"dropped": collection}

            case _:
                raise ValueError(f"unsupported action: {action}")
```

```python
# src/guardmcp/mcp/pipeline.py
from typing import Any
from ..models.domain import Action, DecisionStatus, Request
from ..policy.loader import PolicyLoader
from ..policy.engine import PolicyEngine
from ..risk.engine import RiskEngine
from ..masking.masker import FieldMasker
from ..audit.logger import AuditLogger
from ..approval.store import ApprovalStore
from ..mongo.executor import MongoExecutor


class GuardPipeline:
    def __init__(
        self,
        policy_loader: PolicyLoader,
        policy_engine: PolicyEngine,
        risk_engine: RiskEngine,
        audit_logger: AuditLogger,
        approval_store: ApprovalStore,
        executor: MongoExecutor,
    ) -> None:
        self._policies = policy_loader
        self._policy_engine = policy_engine
        self._risk_engine = risk_engine
        self._audit = audit_logger
        self._approvals = approval_store
        self._executor = executor

    async def run(
        self,
        agent: str,
        collection: str,
        action: Action,
        params: dict[str, Any],
    ) -> dict[str, Any]:
        request = Request(agent=agent, collection=collection, action=action, params=params)

        policy = self._policies.get(agent)
        if policy is None:
            reason = f"no policy configured for agent '{agent}'"
            await self._audit.log(self._audit.build_record(
                agent=agent, collection=collection, action=action.value,
                status="denied", reason=reason, request_id=request.request_id,
            ))
            return {"status": "denied", "reason": reason}

        risk = self._risk_engine.classify(action)
        decision = self._policy_engine.evaluate(request, policy, risk)

        await self._audit.log(self._audit.build_record(
            agent=agent, collection=collection, action=action.value,
            status=decision.status.value, reason=decision.reason,
            risk=risk.value, request_id=request.request_id,
        ))

        if decision.status == DecisionStatus.DENIED:
            return {"status": "denied", "reason": decision.reason}

        if decision.status == DecisionStatus.APPROVAL_REQUIRED:
            approval_req = self._approvals.create(
                agent=agent, action=action.value, collection=collection,
                risk=risk, reason=decision.reason, params=params,
            )
            outcome = await self._approvals.wait_for_decision(approval_req.approval_id)
            if outcome != "approved":
                await self._audit.log(self._audit.build_record(
                    agent=agent, collection=collection, action=action.value,
                    status="denied", reason="approval denied or timed out",
                    risk=risk.value, request_id=request.request_id,
                ))
                return {"status": "denied", "reason": "approval denied or timed out"}
            await self._audit.log(self._audit.build_record(
                agent=agent, collection=collection, action=action.value,
                status="executed_after_approval", risk=risk.value,
                request_id=request.request_id,
            ))

        raw = await self._executor.execute(collection, action, params)
        masker = FieldMasker(policy.mask_fields)
        return {"status": "success", "data": masker.mask_result(raw)}
```

```python
# src/guardmcp/api/approval.py
from fastapi import APIRouter, HTTPException
from pydantic import BaseModel
from ..approval.store import ApprovalStore
from ..approval.models import ApprovalRequest


class DecisionPayload(BaseModel):
    approved: bool


def make_router(store: ApprovalStore) -> APIRouter:
    router = APIRouter(prefix="/approvals", tags=["approvals"])

    @router.get("/pending", response_model=list[ApprovalRequest])
    async def list_pending() -> list[ApprovalRequest]:
        return store.list_pending()

    @router.get("/{approval_id}", response_model=ApprovalRequest)
    async def get_approval(approval_id: str) -> ApprovalRequest:
        req = store.get(approval_id)
        if req is None:
            raise HTTPException(status_code=404, detail="approval not found")
        return req

    @router.post("/{approval_id}/decide")
    async def decide(approval_id: str, payload: DecisionPayload) -> dict:
        req = store.resolve(approval_id, payload.approved)
        if req is None:
            raise HTTPException(status_code=404, detail="approval not found or already resolved")
        return {"approval_id": approval_id, "status": req.status}

    return router
```

```python
# src/guardmcp/__main__.py
import argparse
import threading
import uvicorn
from fastapi import FastAPI
from mcp.server.fastmcp import FastMCP

from .config import Settings
from .policy.loader import PolicyLoader
from .policy.engine import PolicyEngine
from .risk.engine import RiskEngine
from .audit.logger import AuditLogger
from .approval.store import ApprovalStore
from .mongo.client import MongoClient
from .mongo.executor import MongoExecutor
from .mcp.pipeline import GuardPipeline
from .mcp.tools import register_tools
from .api.approval import make_router


def build(settings: Settings) -> tuple[FastMCP, FastAPI]:
    policy_loader = PolicyLoader(settings.policy_path)
    policy_loader.load()

    mongo_client = MongoClient(settings.mongodb_uri, settings.mongodb_database)
    executor = MongoExecutor(mongo_client, settings.max_find_limit)
    audit_logger = AuditLogger(settings.audit_log_path)
    approval_store = ApprovalStore(settings.approval_timeout_seconds)

    pipeline = GuardPipeline(
        policy_loader=policy_loader,
        policy_engine=PolicyEngine(),
        risk_engine=RiskEngine(),
        audit_logger=audit_logger,
        approval_store=approval_store,
        executor=executor,
    )

    mcp = FastMCP("GuardMCP", instructions="Policy-enforced MongoDB gateway. AI proposes. GuardMCP decides.")
    register_tools(mcp, pipeline)

    rest_app = FastAPI(title="GuardMCP Approval API", version="0.1.0")
    rest_app.include_router(make_router(approval_store))

    return mcp, rest_app


def main() -> None:
    parser = argparse.ArgumentParser(description="GuardMCP — Policy-enforced MongoDB MCP server")
    parser.add_argument("--transport", choices=["stdio", "sse", "streamable-http"], default="stdio")
    parser.add_argument("--host", default="127.0.0.1")
    parser.add_argument("--port", type=int, default=8000)
    parser.add_argument("--approval-port", type=int, default=8001)
    args = parser.parse_args()

    settings = Settings(transport=args.transport, host=args.host, port=args.port)
    mcp, rest_app = build(settings)

    if args.transport == "stdio":
        mcp.run(transport="stdio")
    else:
        def run_approval_api() -> None:
            uvicorn.run(rest_app, host=args.host, port=args.approval_port, log_level="warning")

        t = threading.Thread(target=run_approval_api, daemon=True)
        t.start()
        mcp.run(transport=args.transport)


if __name__ == "__main__":
    main()
```

# GuardMCP

> Policy-enforced MongoDB MCP server. AI proposes. GuardMCP decides.

GuardMCP is a secure gateway that exposes MongoDB to MCP-compatible AI clients
(Claude Desktop, Cursor, OpenAI Agents, Gemini) while enforcing YAML-defined policies.

## Quick Start

**1. Create a policy file:**

```yaml
# policies/my-policy.yaml
agent: my-bot
mode: readonly
collections:
  allow:
    - customers
    - orders
mask_fields:
  - email
  - password
```

```bash
GUARDMCP_MONGODB_URI=mongodb://localhost:27017 \
GUARDMCP_POLICY_PATH=policies/my-policy.yaml \
docker run -p 8000:8000 -v $(pwd)/policies:/app/policies guardmcp:latest
```

```json
{
  "mcpServers": {

      "args": ["--transport", "stdio"],

      }
    }
  }
}
```
